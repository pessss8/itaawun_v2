Summary 1 — iTa'awun Project Log
iTa'awun is a centralized web platform for CFS IIUM students, built to replace fragmented WhatsApp-based campus services with a regulated, admin-monitored marketplace. It targets the underprivileged student demographic (B40/Zakat recipients) and is grounded in the Islamic principle of ta'awun — mutual cooperation.
The project started in early March 2026 with a 12 April deadline for completion, IPO application, and user onboarding. Development was repeatedly delayed but ultimately compressed into a final weekend sprint (4–5 April).
Tech stack: HTML/CSS/Vanilla JS frontend hosted on GitHub Pages, Supabase as the free backend (PostgreSQL database, authentication, REST API). No server, no paid services.
What was built across this conversation:
The full authentication flow — index.html (login), signup.html (registration with full name and student ID), forgot.html (Supabase password reset email), and session-based page protection on every page.
The core service pages — find.html (browse 12 hardcoded service listings with filter, search, Contact modal, and a new Book button that writes to Supabase), create.html (vendor service listing form), home.html (main dashboard with hover dropdown and working logout), and profile.html (reads real user data from the profiles table).
The bookings system — bookings.html with two tabs: student view (order history, cancel, rate) and vendor view (incoming requests, accept/reject/mark done). Rating modal built in — 1 to 5 stars with comment, writes to the ratings table.
The admin dashboard — admin.html with four tabs: vendor approval queue, all users with CGPA input, all bookings view, and flagged reports. Stat cards at the top. Admin-role gate so only admin accounts can access it.
Database: 5 Supabase tables — profiles, services, bookings, ratings, reports. Row Level Security not yet applied as of this conversation.
Key bugs fixed along the way: ES module imports breaking on file:// (replaced with CDN global script tag), supabase.createClient() result not being saved to a variable, script blocks placed above <body>, logout bouncing back to home due to session not being cleared properly (fixed with signOut() + window.location.replace()), and profile page showing hardcoded placeholder data instead of real user info.
Remaining before deadline: RLS policies on all 5 Supabase tables, end-to-end testing on the live GitHub Pages URL, IPO application documents (abstract, innovation statement, social impact statement, screenshots), and onboarding 3 to 5 real student testers.
Repository: github.com/pessss8/itaawun (private). Previously prototyped at ipanboy.github.io/testitaawun.
